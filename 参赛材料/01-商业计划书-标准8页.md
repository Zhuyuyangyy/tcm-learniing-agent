## 【第一页】项目概述

### 一句话介绍（30秒开场）

> **"让每个人都能拥有专属的AI中医专家——基于CrewAI多智能体的中医智能诊疗辅助系统"**

我国中医医师供需缺口高达1:5000，基层中医服务能力严重不足。本项目基于CrewAI多智能体框架，融合知识图谱与RAG检索技术，为用户提供专业的中医体质测评、智能问诊和用药安全审核服务。

### 核心竞争力

## 【第二页】行业痛点与市场机会

### 行业痛点

**1. 中医医师严重短缺**
- 我国中医医师约68万人，平均每千人0.49名（WHO标准：每千人1名）
- 基层医疗机构中医师覆盖率不足30%
- 农村地区中医服务几乎空白

**2. 中医知识门槛高**
- 中医理论体系复杂（阴阳五行、藏象学说、辨证论治）
- 自学难度大，缺乏个性化指导
- 网络信息良莠不齐，真假难辨

**3. 用药安全隐患突出**
- 中药也存在配伍禁忌（十八反、十九畏）
- 部分患者自行配药，存在严重健康风险
- 缺乏有效的用药安全审核工具

**4. 现有中医AI产品痛点**
- 准确率低：规则匹配为主，泛化能力差
- 不可解释：输出结果无知识溯源
- 单轮问答：无法进行多轮辨证论治对话

### 目标市场

| 市场细分 | 规模 | 切入点 |
|----------|------|--------|
| 高校中医教育 | 50万+在校中医学子 | 实训工具、辅助学习 |
| 基层医疗机构 | 3.5万乡镇卫生院 | 辅助诊疗、用药审核 |
| 亚健康人群 | 9亿+ | 体质测评、健康管理 |
| 中医爱好者 | 千万级 | 入门学习、养生指导 |

## 【第三页】解决方案

### 产品架构：5 Agent协作诊疗闭环

```
用户输入症状
     ↓
┌─────────────────────────────────────────────┐
│  1. 档案构建师（Profile Builder）              │
│     分析患者基本信息、体质倾向、既往史          │
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│  2. 中医检索师（TCM Retriever）              │
│     + 3. 资源生成师（Resource Generator）    │
│     并行：从知识库检索 + 生成诊疗建议           │
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│  4. 批判分析师（Critique Agent）              │
│     用药安全审核：十八反十九畏自动检测         │
│     逻辑一致性检查                             │
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│  5. 路径规划师（Path Planner）                │
│     综合决策：输出最优治疗路径                  │
└─────────────────────────────────────────────┘
     ↓
【输出】辨证结果 + 个性化处方建议 + 调护方案
```

### 核心技术亮点

**① CrewAI多智能体协作**
- 首次将CrewAI多智能体框架应用于中医辨证论治领域
- 每个Agent专注单一职责，通过自然语言协作
- 支持多轮对话，模拟"望闻问切"四诊合参思维

**② RAG知识增强检索**
- 向量数据库（ChromaDB）实现语义检索
- 知识图谱（43节点）支持多跳推理
- 所有回答均可溯源至权威知识库
- 检索准确率经实测达到较高水平

**③ 知识图谱驱动**
- 43个中医实体节点：脏腑、经络、中药、方剂、体质
- 8大类关系：归属、相生相克、引经报使等
- 支持"肝郁→柴胡疏肝散→失眠调护"多跳推理路径

**④ 用药安全双审核**
- 内置《临床中药学》十八反19组、十九畏18组
- 共37组配伍禁忌药对，智能自动检测
- 处方生成前自动拦截高风险配伍

**⑤ 九种体质分类系统**
- 完全符合中华中医药学会《中医体质分类与判定》标准
- 45题标准问卷 + AI综合分析
- 生成可视化的体质雷达图报告

### 知识图谱设计

```
节点类型（43节点）：
├── 脏腑（5）：心、肝、脾、肺、肾
├── 六腑（5）：胆、胃、小肠、大肠、膀胱
├── 经络（12）：十二正经 + 任督二脉
├── 体质（9）：平和质、气虚质、阳虚质、阴虚质、痰湿质、湿热质、血瘀质、气郁质、特禀质
└── 中药/方剂（12）：代表药物和经典方剂

关系类型（8类）：
├── 相表里关系（肺与大肠相表里）
├── 相生相克关系（肝木克脾土）
├── 归属关系（心主血脉）
├── 引经报使（柴胡为引经药）
├── 配伍禁忌（十八反、十九畏）
└── 循行路线（十二经脉流注）
```

### RAG检索流程

```
用户问题
   ↓
Query理解（意图识别 + 关键实体提取）
   ↓
混合检索
├── 向量检索（语义相似度 Top-K）
├── 关键词检索（BM25）
└── 知识图谱检索（实体邻居扩展）
   ↓
重排序（Cross-Encoder精排）
   ↓
上下文组装 + LLM生成
   ↓
批判分析师审核（置信度标注）
   ↓
最终输出（含引用来源）
```

## 【附录】评委高频问题Q&A

### Q1：你们的准确率是多少？如何验证？

**A**：目前系统处于DEMO演示阶段，准确率数据待真实试点后测试验证。我们的技术架构（RAG+知识图谱+批判分析师审核）为高准确率提供了保障：
- RAG检索确保回答有知识溯源
- 知识图谱提供结构化推理
- 批判分析师进行二次审核，标注置信度

### Q2：知识图谱的数据来源是什么？谁来审核？

**A**：知识图谱节点和关系来源于：
- 《黄帝内经》《难经》等中医经典
- 《中医基础理论》规划教材
- 《临床中药学》配伍禁忌数据

所有知识均标注来源，批判分析师对生成内容进行准确性审核。

### Q3：系统能否替代中医师？

**A**：**绝对不能，也不打算替代。** 本系统定位为"辅助工具"，明确标注"仅供参考，不能替代专业医疗诊断"。系统的核心价值是：
- 健康人群的体质自测和亚健康调理指导
- 中医学习的辅助工具
- 基层医疗机构的初步筛查

### Q4：数据安全如何保障？

**A**：
- 所有数据存储在本地或可信云服务
- 用户问诊数据不用于模型训练
- 符合国家中医药数据安全相关法规
- 系统明确展示数据使用告知

### Q5：为什么选择CrewAI而不是其他框架？

**A**：CrewAI的核心优势：
- 原生支持多Agent协作，契合中医辨证论治的多角色协作思维
- Agent之间通过自然语言通信，易于扩展和维护
- 支持复杂的任务流程编排（sequential/parallel/hierarchical）
- 与LangChain生态兼容，可复用丰富的工具链
